Passion project


Three.js

Voor het passion project mocht ik aan de slag gaan met een eigen gekozen onderwerp. Normaal was dit bedoeld om te focussen op welke leerdoelen je nog moest behalen, bij mij was development mijn grootste zorgen, omdat ik weinig met David had gecommuniceerd. Ik wilde dit daarom zo snel mogelijk verbeteren om mijn leeruitkomsten aan te tonen. Ik heb toen gekozen om te werken aan three.js. Dit is een javascript libarary waar ik later nogal moeite mee kreeg, vanwege de complexe code die hierachter zat. Ik heb me daarom ook vooral gericht op de communicatie tussen 3D en web.

Aanpak

In het begin ben ik gaan kijken naar mijn leeruitkomsten. Waar ik al gelijk aan dacht was development, omdat ik daar nog veel tekort aan kwam. Ik kwam vervolgens met het idee om een website te maken voor iemand die me had benaderd om dit te doen. Het ging om een bedrijf dat gespecialiseerd was in het bouwen van zonnepanelen. Dit plan had ik voorgelegd aan Pieter en kreeg vervolgens te horen dat dit niet een goed idee was. Het had er mee te maken dat het maar basis HTML, CSS en misschien wat Javascript als eindresultaat waarschijnlijk kwam. Na wat nadenken kwam ik tot de conclussie dat Pieter gelijk had en begon ik van plan te veranderen. .

Uitvoering

Om dit allemaal te kunnen realiseren heb ik eerst onderzoek uitgevoerd. Dit heb ik voornamelijk gedaan doormiddel van YouTube. Ik heb gekeken naar vele experts die hierin al waren gespecialiseerd en ieder ander had een andere aanpak. Door de verschillende manieren van aanpak heb ik mijn eigen aanpak gecreerd. Wat ik als eerst deed was inspiratie opdoen door te kijken naar de laatste trends. Als ik iets had gevonden wat ik heel erg mooi of interessant vond, nam ik dat op voor een idee. Vervolgens begon ik dit idee uitvoeren in Spline. Ik gebruikte Spline als een tool waarin je 3D objecten kan creeren en dit kan realiseren in web. Ook koos ik voor Spline, omdat het erg gemakkelijk was te gebruiken en veel van de tutorials die ik keek ook van deze tool gebruik maakten. Als laatst als ik mijn object had gemaakt, importeerde ik de code via een embeded code in mijn HTML code. Dit was een kwestie van kopieren en plakken.

Nadat ik met spline had gewerkt dacht ik dat dit te gemakkelijk was en wist niet precies waar mijn three.js code naartoe zou gaan en of die er uberhaupt was. Dit was nattuurlijk niet het geval, omdat ik alleen een embeded code had gekopierd en geplakt van Spline zelf. Dit wilde zeggen dat mijn code niet geintegreerd was, maar dat ik 2 source codes tot 1 maaktte. Met three.js was dit niet het geval, omdat je vanuit de code zelf werkt tot 1. Tot slot heb ik mijn eindproject uitgevoerd op mijn portfolio. Ik dacht dat het wel een extra toevoeging zou zijn om zo mijn portfolio aantrekkelijker te maken. Ik heb met behulp van een expert veel informatie opgedaan over three.js. De persoon die me hielp heeft eerst de basis uitgelegd en me geholpen met wat three.js inhoudt. Ik heb expres zo een iemand ingeschakeld om zo meer kennis op te doen, omdat ik nog geen voorkennis hierover had.

Na wat uitleg ben ik aan de slag gegaan met mijn portfolio om wat animaties toe te voegen wat eigenlijk vrij gemakkelijk was. Ik heb namelijk gebruik gemaakt van wat timers in de code die afgaan wanneer je voorbij een bepaald punt langs scrollt. Bij het 3D object zelf was er wat meer werk voor nodig. Ik heb JQuery gebruikt voor de interactie op de website en onder andere voor het bewerken van het DOM en CSS en interactie met de webserver. Ook heb ik de cubus gevormd met CSS. Threejs is de Javascript 3D webgl-bibliotheek die de 3D-graphics van de site verwerkt met de kubussen die de animaties aandrijven en het interactief maakt. Dit is ook een open source-bibliotheek. Mijn eindproject was dus ook mijn portfolio.
